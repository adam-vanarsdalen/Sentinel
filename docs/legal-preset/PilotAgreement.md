# SentinelLaw Pilot Agreement (Template)

This is a plain-language pilot agreement template for a 60‑day SentinelLaw pilot. It is not legal advice.

## 1) Pilot term (60 days)

- Pilot start date: ____________________
- Pilot end date (60 days after start): ____________________

## 2) As‑is pilot / no SLA

During the pilot, SentinelLaw is provided **as‑is** for evaluation purposes.

- No uptime or performance SLA during the pilot.
- Features may change during the pilot.
- The firm should not rely on SentinelLaw as its sole control for any high‑risk workflow.

## 3) What the firm agrees to do

The firm agrees to:

- Use SentinelLaw with **at least one real AI workflow** (e.g., drafting, contract review, intake summaries) during the pilot.
- Keep access credentials and API keys secure and limit access to authorized users.
- Provide honest feedback at the end of the pilot, including:
  - what worked,
  - what did not work,
  - key requirements for production adoption.

## 4) What SentinelLaw agrees to do

SentinelLaw agrees to:

- Provide access to the SentinelLaw gateway and admin console for the pilot term.
- Respond to **critical issues** reported by the firm within **48 hours** (best effort during the pilot).
- Keep the firm’s data confidential and use it only to operate and support the pilot.

## 5) Data handling (summary)

SentinelLaw is designed to minimize stored content by default:

- Stores audit metadata, hashes, risk flags, and token counts.
- Does **not** store raw prompts/responses by default.
- Optional redacted snippets can be enabled by firm policy.

See [`../DataHandling.md`](../DataHandling.md) for the full data handling description and defaults.

## 6) Post-pilot path and pricing

If the firm chooses to continue after the 60‑day pilot:

- Commercial pricing and plan details will be documented separately.
- Any post-pilot subscription requires written agreement before the pilot ends.
