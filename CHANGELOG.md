# Changelog

All notable changes to this project will be documented in this file.

The format is based on Keep a Changelog and this project follows semantic versioning conventions for tagged releases.

## [0.1.0] - 2026-04-14

### Added

- Preset-driven product architecture (`general`, `legal`, `finance`, `healthcare`)
- Shared terminology mapping across backend and frontend surfaces
- Multi-org demo seeding with industry-specific scenarios
- Canonical role model (`org_admin`, `compliance_admin`, `operator`, `reviewer`, `auditor`)
- Sentinel-first product framing with SentinelLaw preserved as a legal preset
- Standardized core documentation set for public GitHub publication

### Changed

- Default product identity now presents Sentinel as the shared platform
- Shared policy/risk language generalized beyond legal-only wording
- Repo structure cleaned for canonical root operation in `Sentinel.1`

### Notes

- Existing legal-oriented docs remain as supplemental reference material.
- Release validation workflow remains CI-backed via `.github/workflows/ci.yml`.
